HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Content elements within the body includes:
    • headings (<h1> to <h6>)
    • paragraphs (<p>)
    • images (<img src="path" alt="note of image contents" width="pixel number height="pixel number">)
    • hyperlinks (<a href="url">)
    • tables(<table>,<tr> - row, <th> - header, <td> - cell )
    • lists (unordered <ul> and ordered <ol>, <li> - list item)
    • and more!
  • The body can be further separated into sections using <div> (a division or section in an HTML document which contains HTML elements to be styled with CSS or JS by using a class or id attribute; Any type of content can be put in the <div> tag and browseers always place a line break before and after this element) or <span>(an inline container used to mark up a part of plain text or a part of a document; it's easily styled by CSS or manipulated with JS using class or id attribute)
  • The footer element defines the footer for a document or section and typically contains authorship information, copyright information, contact information, sitemap, back to top links, and/or related documents. There can be multiple footer elements.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • git status: checks what branch we are currently on
  • git check out -b branch-name: creates a new branch and switches to it
  • git config --global user.name "[firstname lastname]": set a name that is identifiable for credit when reviewing version history
  • git config --global user.email "[valid-email]": set an email address that will be associated with each history marker
  • git config --global color.ui auto: set automatic command line coloring for Git for easy reviewing
  • git init: initialize anexisting directory as a Git repository
  • git clone [url]: retrieve an entire repositoory froom a hosted location via url
  • git add [file]: add a file as it looks now to your next commit stage
  • git add -A: commit all files in the woorking branch
  • git reset [file]: unstage a file while retaining the chagnes in working directory
  • git diff: diff of what is changed but not staged
  • git diff --staged: difff of what is staged but not yet commited
  • git commit -m "[descriptive message]": commit your staged content as a new commit snapshot
  • git branch: list your branches. a * will appear next to the currently active branch
  • git branc [branch-name]: create a new branch att the current commit
  • git checkout: switch to another branch and check it out into your working directory
  • git merge [branch]: merge the specified branch's history and check it out into your working directory
  • git log: show all commits in the current branch's history
  • git log branchB..branchA: show the commits on branchAthat are not on branchB
  • git show [SHA]: show any object in Git in human-readable font
  • git rm [file]: delete the file from project and stage the removal for commit
  • git mv [existing-path] [new-path]: change an existing file path and stage the move
  • git log --stat -M: show all commit logs with indication of any paths that moved
  • logs/
    *.notes
    pattern*/: save a file with desired patters as .gitignore with either direct string matches or wildcard globs.
  • git config --global core.excludesfile [file]: system wide ignore pattern ffor all local repositories
